# 🔍 通用受控财务报表复核系统

> 📌 版本：0.1.0（骨架搭建完成，核心框架已就绪，正在逐步完善具体功能）

---

## 📋 这个系统是做什么的？

想象一下：你有一大堆财务报表（比如资产负债表、利润表、现金流量表），还有厚厚的附注说明。你需要仔细核对每一笔数字——报表上的金额和附注里的是不是一致？加总有没有算错？公式有没有漏行？

如果靠人眼一个个看，几十个科目、上百个数字，看花了眼也难免漏掉几个。

**这个系统就是帮你做这件事的。** 它像一个极其认真、不怕累、不偷懒的"数字审计员"，把报表和附注里每一个该查的地方都查一遍，发现问题就记录下来，最后给你一份清清楚楚的问题清单。

而且，它不是随便看看就完事——每发现一个问题，都会同时记录"这个问题在哪个文件的哪个位置""差异是多少""判断理由是什么"，让你可以随时回头核实。

## 📥 怎么安装？

### 第一步：下载系统文件

把整个项目文件夹下载到你的电脑上。如果你用的是 GitHub，可以点"克隆或下载"按钮。

### 第二步：安装 Python 运行环境

1. 去 Python 官网下载安装包
2. 安装时一定要勾选"把 Python 加到路径"这个选项
3. 安装完成后，打开命令行（按键盘的 Win 键，输入"cmd"，回车），输入：python --version，如果显示版本号就说明装好了

### 第三步：安装系统依赖

1. 打开命令行（按 Win 键，输入 cmd，回车）
2. 用命令行进入项目文件夹，输入：cd 加上项目文件夹路径，回车
3. 输入：pip install -e ".[test]"，回车，等待安装完成

> 💡 这一步是在安装系统运行所需的各种"零件"，就像买了一台新电脑要装驱动程序一样。安装一次就行，以后不用重复装。

### 第四步：确认安装成功

在命令行输入：python -m pytest tests/unit/test_package.py -v

如果显示"1 passed"，恭喜你，安装成功了！🎉

## 🚀 怎么使用？

### 方式一：通过人工智能助手使用（推荐）👁️

这是最简单的方式。你只需要：

1. 打开你常用的人工智能助手（比如 Trae、Codex 等）
2. 告诉它："请帮我打开受控财务报表复核系统，项目路径是 xxx"
3. 系统会自动开始工作：读取文件 → 检查数字 → 发现问题 → 生成报告
4. 你只需要等它跑完，然后去看结果

> 💡 系统会自动分配两个"人工智能审计员"分别独立检查，就像安排两个不同的审计师各自独立看一遍，然后对比他们的结论。如果两人意见不一致，还会安排第三个人再看。

### 方式二：通过网页界面使用 🌐

1. 在命令行输入：python -m controlled_review.web.app，回车
2. 打开浏览器，地址栏输入：http://localhost:8000
3. 你会看到一个网页，上面有几个页面：
   - 📝 **创建项目**：选择你要核对的报表文件和附注文件，点击创建
   - 🔗 **对应关系确认**：系统会自动猜出报表项目和附注的对应关系，你可以确认或跳过
   - 📊 **进度查看**：看看检查到哪一步了，发现了多少问题
   - 📋 **结果查看**：检查完成后，在这里看所有发现的问题

### 方式三：通过命令行使用 💻

如果你习惯用命令行，可以直接输入命令：

**打开项目：**

**查看进度：**
> python -m controlled_review.interfaces.cli project_progress "{\"project_id\": \"default-project\"}"

**生成最终报告：**
> python -m controlled_review.interfaces.cli project_finalize "{\"project_id\": \"default-project\"}"

## ⚙️ 系统是怎么工作的？

你可以把整个系统想象成一个"审计工厂"，有明确的流水线：

### 🔧 第一步：接收材料（输入冻结）

你把报表文件和附注文件交给系统。系统会：
- 给每个文件拍一个"指纹"（一种独一无二的数字编码，就像人的指纹一样）
- 以后任何时候都会检查文件有没有被偷偷改过——如果指纹变了，说明文件被动过手脚，系统会立刻停止工作并报警

### 📖 第二步：读懂材料（文件解析）

系统会逐个读取你的文件：
- **Excel 表格**：读取每个单元格的数字、公式、格式，连隐藏的行列都能发现
- **Word 附注**：读取正文、表格、批注、修改痕迹，连脚注和隐藏文字都不放过
- 如果有附注是 Excel 格式的，也能处理

### 🧮 第三步：机器检查（自动核对）

系统会自动做这些检查（不依赖人工智能，纯粹是机器计算）：
- 加总算对了没有——附注里各项明细加起来等于不等于合计数
- 报表平衡不平衡——资产等于不等于负债加所有者权益
- 公式有没有漏行——比如求和公式少包含了一行数据
- 单位统一不统一——报表用"千元"但附注用"元"这种问题
- 本期上期有没有写反
- 有没有被硬编码的数字覆盖了公式

### 🔍 第四步：人工智能判断（语义复核）

机器检查完后，有些事情需要"理解能力"才能判断，比如：
- 科目名称不同但实际意思一样（"应收账款"和"应收款项"）
- 披露内容是否完整
- 差异原因是否合理

这些交给人工智能来做。系统会：
- 把每个需要判断的项目打包成小任务
- 每个任务里偷偷混入一个"测试题"（故意改错一个数字），看人工智能能不能发现
- 如果没发现测试题里的错误，说明人工智能可能在偷懒，整组结果作废重来

### 👥 第五步：双重核对（独立复核）

- 第一个"审计员"检查完后，第二个"审计员"会重新独立检查一遍
- 第二个人看不到第一个人的结论，必须自己从头看
- 如果两人结论一样，通过
- 如果两人结论不一样，换新人重做
- 如果三轮都不一样，标记为"专业分歧"，不再自动重试

### 📦 第六步：生成报告（输出）

所有检查完成后，系统会生成 6 个文件：

| 文件名 | 里面是什么 |
|---|---|
| 📋 复核问题清单.xlsx | 所有发现的问题，每个问题一行，包含位置、差异、理由 |
| 📑 逐项复核台账.xlsx | 每一个该查的地方都列出来，证明没有漏查 |
| ❓ 未确认事项.xlsx | 有些问题拿不准的，单独列出来让你人工判断 |
| 📝 复核总结.docx | 一份总结报告，适合打印归档 |
| 🔗 证据索引.html | 在浏览器里打开，点任何问题都能跳到原始文件位置 |
| 🧾 完成回执.json | 机器可读的完成凭证，包含所有文件指纹和统计信息 |

> ⚠️ 重要：系统从头到尾不会修改你的原始文件。所有操作都在副本上进行，原件一个字都不会动。

## 🎚️ 两种工作模式

系统有两种模式，你可以根据需要选择：

### 🏃 经济模式（快）

- 机器全部检查 + 第一轮人工智能复核
- 只对发现问题的部分做第二轮独立复核
- 普通项目大约半小时完成
- 适合时间紧、初步筛查

### 🔬 严格模式（慢但更可靠）

- 机器全部检查 + 所有项目都做两轮独立复核
- 普通项目大约一到三小时
- 适合正式报告、审计底稿

> 💡 两种模式都会做机器全量检查和隐藏测试，区别只是第二轮独立复核的范围不同。不是"严格模式检查得更多"，而是"严格模式所有项目都有第二个人复查"。

## 🛡️ 安全保障

- 🔒 **原件只读**：系统从设计上就不提供修改原始文件的功能
- 🔐 **证据防伪**：每条证据都有数字签名，伪造的证据会被拒绝
- 🚫 **隔离工作**：两个"审计员"互相看不到对方的结论，防止抄答案
- 📛 **域名白名单**：查询官方法规时只从财政部、证监会等官方网站获取，不会随便信网上的内容
- 🧪 **隐藏测试**：每个任务都偷偷放一个"假错题"，偷懒会被抓到

## ❓ 常见问题

**问：系统能代替注册会计师吗？**
答：不能。系统只是辅助工具，帮你发现数字层面的问题。最终的专业判断还是需要注册会计师来做。

**问：系统会修改我的报表吗？**
答：绝对不会。系统从头到尾只读你的文件，不会修改任何原始文件。所有计算都在临时副本上进行。

**问：如果电脑突然关机了怎么办？**
答：没事。系统会把进度存在本地数据库里。重新打开后自动恢复到上次的位置，已经检查完的不会重做。

**问：没有装微软 Office 能用吗？**
答：部分功能可以用（比如文件解析），但 Excel 公式重新计算功能需要 Office。建议装上 Office 以获得完整功能。

**问：支持苹果电脑吗？**
答：不支持。系统是专门为 Windows 设计的，因为依赖微软 Office 的自动化能力。

**问：系统会泄露我的财务数据吗？**
答：系统只在你本地运行，不会向互联网发送任何数据。查询官方法规时只访问财政部、证监会等官方网站，不会把你的报表数据发出去。

### 关键设计决策

**1. 单写核心（ADR-0001）**
所有平台共用一套本地核心，MCP 适配器和 CLI 入口都调用同一个 AppService，不直接写数据库。跨进程互斥通过 writers 表 + BEGIN IMMEDIATE 事务实现。

**2. 状态独立于聊天（ADR-0002）**
项目进度保存在 SQLite，不依赖聊天记录。平台切换、进程退出、电脑重启后可从安全断点恢复。租约过期自动回收：reviewer 回 pending，verifier 回 reviewer_passed。

**3. 原件只读（ADR-0003）**
系统不提供修改原始文件的工具。所有 Office 操作在临时副本上进行。每次取证和最终输出前重新校验 SHA256 摘要。

**4. 双视图 XLSX 读取**
同时加载 data_only=False（公式）和 data_only=True（缓存值）两个视图，检测公式与缓存值不一致。提取隐藏行列、合并区域、命名区域、外部链接、打印区域。

**5. HMAC-SHA256 证据签名**
每条证据的签名 payload 包含七元组（assignment_id、role、target_id、file_sha256、node_id、context_sha256、nonce）。验证时按 存在性 → target_id → assignment/role → 签名 顺序校验，使用 hmac.compare_digest 常量时间比较。

**6. 隐藏测试不透明标识**
Canary 的 public_id 使用 secrets.token_hex(8) 随机生成，工作者无法从编号识别隐藏测试身份。7 种单字段变异（swap_period / change_scope / change_unit / change_currency / replace_note_number / perturb_amount / replace_account_name），漏检导致整组真实结论作废。

**7. 严格模式门禁**
strict 模式无隔离子代理且无备用模型时返回 strict_unavailable，拒绝伪造运行。verifier 使用全新隔离会话，payload 不包含 reviewer_result / reviewer_reason / reviewer_evidence_ids。

**8. OFFICIAL_HOSTS 白名单**
官方依据查询仅访问以下域名：mof.gov.cn、kjs.mof.gov.cn、csrc.gov.cn、sse.com.cn、szse.cn、gov.cn、sasac.gov.cn。网络失败时返回 official_unconfirmed，不阻塞内部检查。

在 Trae IDE 中开发时，git commit 会被沙箱标记为高风险命令。所有 git 操作需通过 Python subprocess 包装执行：

python -c "import subprocess; subprocess.run(['git', 'commit', '-m', '...'], cwd=r'...', capture_output=True, text=True)"
